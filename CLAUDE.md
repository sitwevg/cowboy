# Cowboy — учебный проект

## О проекте
Мобильная игра по мотивам DOS-игры про ковбоя. Учебный проект.

## Платформа
- **Фреймворк:** Expo (React Native)
- **Язык:** JavaScript / TypeScript
- **Сборка:** EAS Build (expo.dev, free tier)
- **Репозиторий:** GitHub

## Жанр
Текстовый квест с картой. Grid-based движение, случайные события, управление ресурсами.

## Структура проекта
- `project.md` — геймдизайн и механики
- `src/` — исходный код
- `assets/` — графика и иконки

## Контекст разработки
- Разработчик изучает мобильную разработку
- Опыт: PWA (трекер привычек), Telegram Mini App, Unity (прототип)
- Новое в этом проекте: Expo / React Native, мобильная разработка без Unity

## Окружение разработки

### Node.js
- Установлен через **nvm**
- Версия: **v20.20.0**
- Активация перед работой: `source ~/.nvm/nvm.sh && nvm use v20.20.0`

### Expo SDK
- Версия: **54** (совместима с Expo Go v54.0.6 на телефоне)
- Папка проекта: `app/`
- Зависимости: expo ~54.0.0, react 19.1.0, react-native 0.81.5, expo-status-bar ~3.0.9
- Установка зависимостей: `npm install --legacy-peer-deps`

### Тестирование на телефоне
- Приложение: **Expo Go v54.0.6** (Android, установлено из Google Play)
- Телефон и ноутбук в одной WiFi сети — тоннель не нужен
- Запуск сервера: `cd ~/Z/cowboy/app && npx expo start`
- Сканировать QR-код в Expo Go — телефон обновляется автоматически при изменении кода
- Запасной вариант (другая сеть): `npx expo start --tunnel` (ngrok установлен глобально)

### Оригинальная игра
- Файлы: `original/WAY.EXE`, `original/WHAT_NEW.TXT`, `original/strings_ru.txt`, `original/game_text.txt`
- `original/game_data.md` — числовые данные (цены, механики), извлечённые из бинарника
- Запуск в DOSBox: конфиг `~/.dosbox/dosbox-0.74-3.conf`, кодировка `keyb ru 866` в autoexec
- Быстрый запуск: лаунчер на рабочем столе (`~/games-launcher.sh`) → "WAY: Ковбой (DOS, 1999)"
- `game_text.txt` — все русские строки из бинарника (CP866), используется как эталон текстов
- Скриншоты из DOSBox сохраняются в `~/Изображения/` → перемещать в `original/screenshots/`
- Прочитанные скриншоты перекладывать в `original/screenshots/done/`
- Новые (непрочитанные) всегда лежат в корне `original/screenshots/`
