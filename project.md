# Cowboy — План разработки

Мобильный порт DOS-игры WAY.EXE (1999, Яценко Сергей).
Цель: полноценная игра на Android + веб-версия (Vercel).
Источник правды по механикам: `original/game_data.md`.
Скриншоты оригинала для UI: `original/screenshots/`.

---

## Стек

- **Expo SDK 54** / React Native + Expo Web (один код — Android и браузер)
- Язык: JavaScript
- Запуск локально: `cd app && npx expo start`
- Тест Android: Expo Go (одна сеть) или `--tunnel`
- Тест веб: `npx expo start --web` → localhost
- Деплой на Vercel: `npx expo export --platform web` → папка `dist/` → Vercel

---

## UI/UX решения

| Вопрос | Решение |
|---|---|
| Стиль | DOS-стиль: чёрный фон, моноширинный шрифт, ASCII-рамки ╔═╗, цвета оригинала (cyan заголовки, white данные, red/yellow акценты) |
| Ориентация | Альбомная (landscape) — фиксированная |
| Карта | Viewport (видна часть), зажать карту + перетащить → pan; кнопки zoom (+/−) в левом столбике |
| Навигация по карте | Кнопки ↑↓←→ справа снизу поверх карты, draggable (long press + drag) |
| Меню | Кнопка «МЕНЮ» в углу экрана |
| Диалоги (1/2/3) | Кнопки вместо цифр |
| Enter (листать) | Тап по экрану или кнопка «Далее» |
| Ввод числа | Числовая клавиатура (цена вола, кол-во товара) |

## Layout-ы экранов (три разных)

```
A. MapScreen       — карта с fog of war, zoom, nav-кнопки, верхняя строка статуса
B. InventoryScreen — [ВЕЩЕВОЙ МЕШОК | ОКОЛО ВАС] + [ОКНО СООБЩЕНИЙ]
                     Используется: утро, торговля, лут после боя
C. EventScreen     — [ВАШИ ДАННЫЕ | ВАШИ УМЕНИЯ] + [ОКНО СООБЩЕНИЙ | мини-карта]
                     Используется: события, бой, все игровые взаимодействия
```

Дополнительно:
```
D. SplashScreen    — чёрный, зелёный текст по центру (заставка)
E. MenuScreen      — чёрный, рамка МЕНЮ по центру (список пунктов)
F. LevelUpScreen   — текст прокачки (можно как EventScreen с особым содержимым)
G. EndScreen       — победа или поражение
```

## Архитектура

```
app/src/
  engine/
    gameState.js   — состояние игры (всё в одном объекте)
    map.js         — генерация карты, туман войны, движение
    events.js      — генератор событий дня/ночи, рандом
    combat.js      — пошаговый бой, state machine
    trading.js     — диалоги торговли, цены, лут
    skills.js      — эффекты умений, применение при действиях
  screens/         — React Native экраны (A–G выше)
  components/
    DosBox.js      — ASCII-рамка, переиспользуемый контейнер
    MessageWindow.js — окно сообщений с накоплением строк
    MiniMap.js     — мини-карта справа снизу
  data/
    constants.js   — BASE_HP, SPREAD, цены, веса, скорости
    strings.js     — все фразы из game_data.md (точная орфография)
    tables.js      — таблицы точности, вероятностей событий
```

Логика в engine/ не зависит от UI — тестируемая отдельно.

## Игровой цикл (flow одного дня)

```
MapScreen
  → игрок делает шаг
    → [если новая ячейка = таверна/салун/индейцы] → InventoryScreen (торговля)
    → иначе: утро

УТРО (InventoryScreen)
  → выбор рациона (1/2/3)
  → списание еды
    → [нет еды] → цепочка: сапоги? → вол? → −40 ЖО
  → лечение? «Будем лечится? ЖО= (1-Да,2-Нет)»

ДЕНЬ (EventScreen, 0–4 события)
  → для каждого слота: случайное событие (ничего / волк / антилопа / бандит / шериф / индейцы / болезнь / змея / кактус)

ВЕЧЕР (EventScreen)
  → «Спать (1) или ехать (2)?»

НОЧЬ (EventScreen, 0–1 событие)
  → восстановление ЖО (сон: +10 / +15 с Регенерацией; движение: +5 с Регенерацией)
  → 0–1 случайное событие

→ MapScreen
```

### Фаза 0 — Веб-адаптация и деплой (делается один раз в начале)

**Цель:** с первого дня разработки видеть результат в браузере на Vercel.

#### 0.1 Настройка Expo Web
- [ ] Установить зависимости: `npx expo install react-native-web react-dom @expo/metro-runtime`
- [ ] Проверить: `npx expo start --web` запускается без ошибок
- [ ] Фиксированная альбомная ориентация на вебе: в `app.json` → `"orientation": "landscape"`; на вебе через CSS: `html, body { height: 100%; overflow: hidden; }`

#### 0.2 Vercel
- [ ] `vercel.json` в корне `app/`:
  ```json
  {
    "buildCommand": "npx expo export --platform web",
    "outputDirectory": "dist",
    "framework": null
  }
  ```
- [ ] Подключить репозиторий к Vercel (после создания GitHub-репо)
- [ ] Автодеплой при push в `main`
- [ ] Проверить: сайт открывается, базовый экран отображается

#### 0.3 Адаптация управления для веба
- [ ] Стрелки клавиатуры → движение по карте (web-only, `window.addEventListener('keydown')`)
- [ ] Клавиши 1/2/3 → выбор в диалогах (web-only)
- [ ] Enter → «Далее» в окне сообщений (web-only)
- [ ] Цифры 0–9 + Enter → ввод числа в числовых полях (кол-во товара, цена вола, кол-во мяса) — работает через стандартный `<TextInput keyboardType="numeric">`, на вебе нативная клавиатура
- [ ] Мышь: клик = тап везде (работает автоматически в react-native-web)
- [ ] Drag nav-кнопок на вебе: использовать `onMouseDown/onMouseMove` вместо touch

#### 0.4 Что может отличаться на вебе (учитывать при разработке)
- Моноширинный шрифт: на вебе использовать `font-family: 'Courier New', monospace` явно
- ASCII-рамки могут рендериться иначе — проверять на обеих платформах
- `AsyncStorage` работает в вебе через `localStorage` (встроено в Expo)
- Числовая клавиатура на вебе = обычный `<input type="number">`

### Фаза 3 — События

**Цель:** случайные события дня и ночи.

#### 3.1 Система слотов
- [ ] Количество дневных слотов: случайное 0–4 (из наблюдений)
- [ ] Количество ночных слотов: случайное 0–1
- [ ] Для каждого слота: выбрать событие по вероятности (таблица в `data/tables.js`)
- [ ] Заголовок «Н И К О Г О» при пустом событии, две фразы: «На горизонте ничего не видно» / «Хм.. пока ничего не случилось»

#### 3.2 Пассивные события (без боя)
- [ ] **Змея**: «Вас укусила змея, в следующий раз смотри куда идёшь» → −20 ЖО (мгновенно, без выбора)
- [ ] **Кактус**: «вы наступили на кактус» → −10 ЖО (~2–3% вероятность)
- [ ] **Болезнь**: выбрать тип (насморк −5, грипп −10, желтуха −15), применить урон, показать реплику. Без выбора — просто снимается ЖО

#### 3.3 Скорость по местности
- [ ] Пустыня: 1 день/шаг
- [ ] Зыбучие пески: 2 дня/шаг
- [ ] Горы: 3 дня/шаг
- [ ] Количество дневных слотов умножается на кол-во дней шага

#### 3.4 Умение Иммунитет
- [ ] При событии «болезнь»: шанс заболеть уменьшается на 5% за стак Иммунитета

### Фаза 4 — Бой

**Цель:** полноценный пошаговый бой для всех типов врагов.

#### 4.1 State machine боя
```
ENCOUNTER → PLAYER_TURN → ATTACK_RESOLVE → ENEMY_TURN → (PLAYER_TURN | LOOT | FLEE | DEATH)
```
- [ ] Состояния боя в `combat.js`
- [ ] Начальная дистанция: волк 0м, антилопа/бандит/Билл 50м
- [ ] Меню игрока: «Атакуем, Ура!» / «Лучше-подойти» / «Уходим,уходим»

#### 4.2 Атака игрока
- [ ] Расчёт попадания: таблица точности (дистанция × день/ночь) из `data/tables.js`
- [ ] Урон: random(3,15) для кольта, random(1,7) для ножа
- [ ] Умение Снайпер: +5% к точности на всех дистанциях за стак
- [ ] Умение Доп. урон: +1 к урону за стак
- [ ] Флейвор-реплика перед результатом (из strings.js)
- [ ] Критический урон игрока по волку: «Да ты в глаз зверю попал (второй скушать можно).»

#### 4.3 Ответный ход врага
- [ ] Расчёт попадания врага: своя таблица по типу врага (бандит/волк/Билл)
- [ ] Урон врага по диапазону из `data/constants.js`
- [ ] Умение Живучесть: финальный урон = max(0, урон − живучесть); 0 = успешная атака
- [ ] Критический урон врага: 0.5%, random(0,255)

#### 4.4 Сближение («Лучше-подойти»)
- [ ] Шанс обнаружения — таблица из `data/tables.js`:

  | | 50м день | 50м ночь | 25м день | 25м ночь | 0м день | 0м ночь |
  |---|---|---|---|---|---|---|
  | Антилопа | 45% | 25% | 70% | 45% | 95% | 75% |
  | Бандит | 15% | 5% | 35% | 20% | 65% | 45% |

- [ ] Умение Тихоход: −5% за стак ко всем значениям таблицы
- [ ] Не замечен → снова ход игрока
- [ ] Замечен → враг атакует, затем ход игрока
- [ ] «А ближе некуда» при 0м — ход не теряется
- [ ] **Выстрел = гарантированное обнаружение**: при «Атакуем, Ура!» или перезарядке враг замечает игрока всегда, с любой дистанции — таблица вероятностей больше не применяется

#### 4.5 Перезарядка
- [ ] 6 патронов в барабане, убывают при выстреле
- [ ] При 0 патронах + попытка атаковать: «Уф, увлекся так, что забыл перезарядить, займемся...»
- [ ] Перезарядка: враг всегда делает ход
- [ ] Если патроны кончились навсегда: переключение на нож, «Стоп, осечка вышла, патрончики — ку-ку!»

#### 4.6 Побег
- [ ] Побег игрока («Уходим,уходим»): от волка нельзя («А этот волк приближается»)
- [ ] Побег врага: вероятностный при ≤10 ЖО остатка
  - Волк при побеге: дистанция → 25м → следующий ход 50м → «Цель вне досягаемости»
  - Бандит убегает аналогично
  - Дикий Билл не убегает никогда

#### 4.7 Особенности Дикого Билла
- [ ] Вступление: «Мама, а-а-а! Это Дикий Билл! (Капец тебе мужик). У Билла БОЛЬШАЯ ПУШКА — шли похоронки родственникам.»
- [ ] Билл стреляет первым (до меню игрока)
- [ ] Заклинивание: вместо выстрела «Фу, облом ему – ружье заело» — Билл теряет ход
- [ ] Флейвор перед ходом игрока: «Вытри сопли и заряжай!»

#### 4.8 Антилопа
- [ ] Начинает на 50м, при обнаружении убегает (не атакует)
- [ ] «Вы их спугнули. Стадо поспешно убегает.»
- [ ] При победе: заголовок «Ж И В О Т Н О Е», ввод количества мяса (ограничен весом)

#### 4.9 Лут
- [ ] Бандит/Билл: «Ну что чистим карманы?» Да/Нет → весь лут без проверки веса
- [ ] Животное: «Ну что будем брать мясо и сколько?» → ввод числа, проверка веса
- [ ] Опыт: волк +40, антилопа +30, бандит +50, Билл +100
